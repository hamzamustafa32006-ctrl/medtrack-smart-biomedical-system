Objective
Automatically generate, display, and notify users about upcoming or overdue maintenance events for equipment.
🎯 Key Capabilities
Detect when a piece of equipment is due for maintenance.
Create alerts automatically (daily background job).
Display alerts in-app (color-coded and actionable).
Send optional notifications (email / SMS / push).
Resolve or acknowledge alerts when tasks are completed.
🧱 2. Database Design
New Table: maintenance_alerts
CREATE TABLE maintenance_alerts (
  id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  equipment_id UUID REFERENCES equipment(id),
  facility_id UUID REFERENCES facilities(id),
  location_id UUID REFERENCES locations(id),
  severity TEXT CHECK (severity IN ('Info','Warning','Critical')) DEFAULT 'Info',
  title TEXT NOT NULL,
  message TEXT NOT NULL,
  due_date DATE,
  triggered_at TIMESTAMP DEFAULT NOW(),
  status TEXT CHECK (status IN ('Open','Acknowledged','Resolved')) DEFAULT 'Open',
  acknowledged_by UUID REFERENCES users(id),
  acknowledged_at TIMESTAMP,
  resolved_at TIMESTAMP
);
🧠 3. Alert Logic (Backend)
Step 1: Daily Maintenance Checker (Cron Job)
Runs every night to scan all equipment and generate alerts if needed.
from datetime import date, timedelta
from db import query, execute

def generate_alerts():
    today = date.today()
    rows = query("""
      SELECT id, name, facility_id, location_id, next_due_date
      FROM equipment
      WHERE status != 'Decommissioned'
    """)

    for e in rows:
        days_left = (e["next_due_date"] - today).days
        if days_left == 7:
            severity = "Warning"
            title = f"Maintenance due soon for {e['name']}"
            message = f"{e['name']} at facility {e['facility_id']} is due in 7 days."
        elif days_left <= 0:
            severity = "Critical"
            title = f"Overdue maintenance: {e['name']}"
            message = f"{e['name']} maintenance is {abs(days_left)} day(s) overdue!"
        else:
            continue

        # Avoid duplicate alert
        existing = query("""
          SELECT id FROM maintenance_alerts
          WHERE equipment_id = %s AND status='Open'
        """, [e["id"]])
        if not existing:
            execute("""
              INSERT INTO maintenance_alerts (equipment_id, facility_id, location_id,
                severity, title, message, due_date)
              VALUES (%s,%s,%s,%s,%s,%s,%s)
            """, [e["id"], e["facility_id"], e["location_id"], severity, title, message, e["next_due_date"]])
✅ Runs daily → creates alerts only once per equipment until resolved.
✅ You can use Celery / BullMQ / CronJob to schedule this logic.
🔁 4. Alert Lifecycle
State	Description	Trigger
Open	Newly generated, awaiting review	Created by scheduler
Acknowledged	User marked as seen	User clicks “Acknowledge”
Resolved	Task completed	Linked maintenance task closed
🔔 5. API Endpoints
Method	Endpoint	Description
GET	/alerts	List all open alerts
POST	/alerts/:id/ack	Acknowledge alert
POST	/alerts/:id/resolve	Mark alert as resolved
GET	/alerts?facility_id=&severity=	Filter alerts by facility, severity, or status

🧩 8. Integration with Maintenance Tasks
When a technician completes a maintenance task:
UPDATE maintenance_alerts
SET status='Resolved', resolved_at=NOW()
WHERE equipment_id=:equipment_id AND status='Open';
✅ Ensures no stale alerts remain after task completion.

Data Management Improvements (Backend)
✅ A. Expand the Equipment Data Model
Add richer metadata to make equipment tracking more meaningful.
ALTER TABLE equipment
ADD COLUMN manufacturer TEXT,
ADD COLUMN model TEXT,
ADD COLUMN serial_number TEXT,
ADD COLUMN department TEXT,
ADD COLUMN vendor_name TEXT,
ADD COLUMN purchase_date DATE,
ADD COLUMN warranty_expiry DATE,
ADD COLUMN maintenance_frequency_days INT,
ADD COLUMN calibration_required BOOLEAN DEFAULT FALSE,
ADD COLUMN image_url TEXT;  -- For uploaded photo or QR scan
💡 These fields let you track ownership, maintenance patterns, and device history per department.
✅ B. Add Equipment Status Automation
Automatically update status based on alerts or maintenance tasks.
UPDATE equipment
SET status = CASE
  WHEN id IN (SELECT equipment_id FROM maintenance_alerts WHERE severity='Critical' AND status='Open') THEN 'Under Maintenance'
  WHEN id IN (SELECT equipment_id FROM maintenance_alerts WHERE severity='Warning' AND status='Open') THEN 'Scheduled'
  ELSE 'Active'
END;
💡 This way, the dashboard always shows real-time operational status.
✅ C. Introduce Equipment Audit Trail
Track every change (useful for hospital compliance or biomedical audits).
CREATE TABLE equipment_audit (
  id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  equipment_id UUID REFERENCES equipment(id),
  user_id UUID REFERENCES users(id),
  action TEXT,
  old_data JSONB,
  new_data JSONB,
  timestamp TIMESTAMP DEFAULT NOW()
);
Every time a record is updated, insert a diff entry here.
✅ D. Optimize Queries
Add indexes for faster filtering:
CREATE INDEX idx_equipment_facility ON equipment(facility_id);
CREATE INDEX idx_equipment_status ON equipment(status);
CREATE INDEX idx_equipment_next_due ON equipment(next_due_date);

Objectives
Make equipment data clear, searchable, and relational.
Link each item to its facility, location, vendor, and maintenance status.
Allow quick filtering and sorting by any key attribute.
Present details visually (status colors, icons, QR tags).
🧱 2. Improved Data Model
Your equipment table should be normalized and contextualized.
✅ Updated Schema
CREATE TABLE equipment (
  id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  equipment_id_code TEXT UNIQUE,
  name TEXT NOT NULL,
  type TEXT,
  manufacturer TEXT,
  model TEXT,
  serial_number TEXT,
  facility_id UUID REFERENCES facilities(id),
  location_id UUID REFERENCES locations(id),
  vendor_id UUID REFERENCES vendors(id),
  status TEXT CHECK (status IN ('Active','Under Maintenance','Decommissioned','Pending Installation')) DEFAULT 'Active',
  last_maintenance_date DATE,
  next_due_date DATE,
  criticality TEXT CHECK (criticality IN ('Low','Medium','High')) DEFAULT 'Medium',
  purchase_date DATE,
  warranty_expiry DATE,
  notes TEXT,
  created_at TIMESTAMP DEFAULT NOW(),
  updated_at TIMESTAMP DEFAULT NOW()
);
✅ Relations
Each equipment links to a facility, location, and vendor.
Each has computed status & maintenance cycle.
Add next_due_date so the frontend can color-code urgency.

System Overview
🎯 Goal
Create an integrated environment where users can:
Add, view, update, and organize equipment with full metadata.
Link equipment to facilities, locations, vendors, and maintenance schedules.
Automatically generate and manage alerts when maintenance or calibration is due.
Monitor active alerts and overdue equipment in a centralized Alert Center.
🧱 2. Database Schema
🔹 Equipment Table
CREATE TABLE equipment (
  id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  equipment_code TEXT UNIQUE,
  name TEXT NOT NULL,
  type TEXT,
  model TEXT,
  manufacturer TEXT,
  serial_number TEXT,
  facility_id UUID REFERENCES facilities(id),
  location_id UUID REFERENCES locations(id),
  vendor_id UUID REFERENCES vendors(id),
  purchase_date DATE,
  warranty_expiry DATE,
  criticality TEXT CHECK (criticality IN ('Low','Medium','High')) DEFAULT 'Medium',
  status TEXT CHECK (status IN ('Active','Under Maintenance','Decommissioned')) DEFAULT 'Active',
  last_maintenance_date DATE,
  next_due_date DATE,
  notes TEXT,
  created_at TIMESTAMP DEFAULT NOW(),
  updated_at TIMESTAMP DEFAULT NOW()
);
🔹 Maintenance Alerts Table
CREATE TABLE maintenance_alerts (
  id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  equipment_id UUID REFERENCES equipment(id),
  facility_id UUID REFERENCES facilities(id),
  location_id UUID REFERENCES locations(id),
  severity TEXT CHECK (severity IN ('Info','Warning','Critical')) DEFAULT 'Info',
  title TEXT,
  message TEXT,
  due_date DATE,
  status TEXT CHECK (status IN ('Open','Acknowledged','Resolved')) DEFAULT 'Open',
  triggered_at TIMESTAMP DEFAULT NOW(),
  acknowledged_at TIMESTAMP,
  resolved_at TIMESTAMP
);
🧠 3. Backend Logic
🔹 Daily Maintenance Scheduler (CRON / Background Worker)
Runs daily to check which devices need maintenance soon or are overdue.
from datetime import date
from db import query, execute

def generate_alerts():
    today = date.today()
    rows = query("""
      SELECT id, name, next_due_date, facility_id, location_id
      FROM equipment WHERE status != 'Decommissioned'
    """)
    for e in rows:
        days_left = (e["next_due_date"] - today).days if e["next_due_date"] else None
        if days_left is None: continue

        if days_left <= 0:
            severity = "Critical"
            title = f"{e['name']} maintenance overdue!"
            message = f"{e['name']} in {e['location_id']} is {abs(days_left)} day(s) overdue."
        elif days_left <= 7:
            severity = "Warning"
            title = f"{e['name']} maintenance due soon"
            message = f"Due in {days_left} day(s)."
        else:
            continue

        existing = query("SELECT id FROM maintenance_alerts WHERE equipment_id=%s AND status='Open'", [e["id"]])
        if not existing:
            execute("""
              INSERT INTO maintenance_alerts (equipment_id, facility_id, location_id, severity, title, message, due_date)
              VALUES (%s,%s,%s,%s,%s,%s,%s)
            """, [e["id"], e["facility_id"], e["location_id"], severity, title, message, e["next_due_date"]])
✅ Automatically creates new alerts for due or overdue devices
✅ Prevents duplicates
✅ Easily extendable with vendor warranty reminders

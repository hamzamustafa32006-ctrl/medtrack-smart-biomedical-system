⚙️ 2. Automation Logic — Auto-Status Updates & Smart Tracking
Here’s where your system becomes intelligent 👇
✅ A. Daily Maintenance Status Automation
Automatically change the status of each equipment based on maintenance due dates or alerts.
Example (Node.js cron job, 2 AM daily)
import cron from 'node-cron';
import db from './db.js';
import dayjs from 'dayjs';

cron.schedule('0 2 * * *', async () => {
  const today = dayjs().format('YYYY-MM-DD');

  // Fetch all equipment with active schedules
  const { rows: eqs } = await db.query(`
    SELECT id, next_due_date, status
    FROM equipment
    WHERE status != 'Decommissioned' AND next_due_date IS NOT NULL
  `);

  for (const e of eqs) {
    const daysLeft = dayjs(e.next_due_date).diff(today, 'day');

    let newStatus;
    if (daysLeft <= 0) newStatus = 'Under Maintenance';
    else if (daysLeft <= 7) newStatus = 'Pending Inspection';
    else newStatus = 'Active';

    if (newStatus !== e.status) {
      await db.query(`UPDATE equipment SET status=$1 WHERE id=$2`, [newStatus, e.id]);
    }
  }
});
✅ Automatically switches:
“Active” → “Pending Inspection” → “Under Maintenance” → back to “Active” once resolved
Keeps equipment list always up to date
✅ B. Automated next_due_date Rolling
When a technician marks maintenance complete:
UPDATE equipment
SET
  last_maintenance_date = CURRENT_DATE,
  next_due_date = CURRENT_DATE + (maintenance_frequency_days || ' days')::interval,
  status = 'Active'
WHERE id = $1;
This creates continuous maintenance scheduling without manual date entry.
✅ C. Condition Monitoring (Optional Upgrade)
If you later integrate IoT sensors or manual readings (e.g., pressure, voltage, or calibration logs), you can auto-update condition:
UPDATE equipment
SET condition = CASE
  WHEN usage_hours > 1000 THEN 'Needs Repair'
  WHEN calibration_required AND calibration_date < CURRENT_DATE - INTERVAL '365 days' THEN 'Critical'
  ELSE 'Good'
END
WHERE status = 'Active';

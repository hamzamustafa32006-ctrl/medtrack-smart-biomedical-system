## 🧭 1. Goal

Enable users to:

* Filter equipment by **maintenance status** (Active, Under Maintenance, Overdue, Pending Inspection, etc.)
* Search by **name, model, serial number, or facility**
* Combine both in one request

Example:

```
GET /equipment?status=Under%20Maintenance&search=ventilator
```

---

## 🧱 2. Database Prerequisites

Your `equipment` table should already contain:

```sql
name TEXT,
manufacturer TEXT,
model TEXT,
serial_number TEXT,
facility_id UUID,
status TEXT CHECK (status IN ('Active','Pending Inspection','Under Maintenance','Decommissioned'))
```

✅ Make sure to index searchable fields for faster lookups:

```sql
CREATE INDEX idx_equipment_status ON equipment(status);
CREATE INDEX idx_equipment_textsearch ON equipment USING gin (to_tsvector('english', name || ' ' || model || ' ' || serial_number));
```

## ⚙️ 3. Backend Endpoint (Node.js + Express Example)

### ✅ **Route:**

```js
// GET /equipment?status=&search=
app.get('/equipment', async (req, res) => {
  const { status, search } = req.query;

  let baseQuery = `
    SELECT e.*, f.name AS facility_name
    FROM equipment e
    LEFT JOIN facilities f ON f.id = e.facility_id
    WHERE 1=1
  `;
  const params = [];

  // Filter by maintenance status
  if (status) {
    params.push(status);
    baseQuery += ` AND e.status = $${params.length}`;
  }

  // Text search by name/model/serial
  if (search) {
    params.push(`%${search.toLowerCase()}%`);
    baseQuery += ` AND (LOWER(e.name) LIKE $${params.length}
                     OR LOWER(e.model) LIKE $${params.length}
                     OR LOWER(e.serial_number) LIKE $${params.length}
                     OR LOWER(f.name) LIKE $${params.length})`;
  }

  baseQuery += ' ORDER BY e.name ASC';

  try {
    const { rows } = await db.query(baseQuery, params);
    res.json(rows);
  } catch (error) {
    console.error('Error fetching equipment:', error);
    res.status(500).json({ error: 'Failed to retrieve equipment.' });
  }
});
```

✅ Supports multiple filters
✅ Prevents SQL injection
✅ Works even if filters are not provided

## 🧠 6. Advanced Filtering (Optional Enhancements)

You can add more filters easily by extending the endpoint:

| Filter      | Example Param           | SQL                         |
| ----------- | ----------------------- | --------------------------- |
| Facility    | `?facility=UUID`        | `AND e.facility_id = $x`    |
| Criticality | `?criticality=High`     | `AND e.criticality = $x`    |
| Due Range   | `?dueBefore=2025-12-01` | `AND e.next_due_date <= $x` |

Example:

```
/equipment?status=Active&facility=3b4d...&dueBefore=2025-12-01
```

## 🖥️ 8. Frontend Integration Example

### React Fetch Hook

```js
const fetchEquipment = async (status, search) => {
  const params = new URLSearchParams();
  if (status) params.append('status', status);
  if (search) params.append('search', search);
  
  const res = await fetch(`/equipment?${params.toString()}`);
  return res.json();
};
```

### Search & Filter UI


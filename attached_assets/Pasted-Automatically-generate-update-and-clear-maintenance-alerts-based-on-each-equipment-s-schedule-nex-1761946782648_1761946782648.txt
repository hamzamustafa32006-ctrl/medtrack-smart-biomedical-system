Automatically generate, update, and clear maintenance alerts based on each equipment’s schedule (next_due_date).
The system should:
Check equipment daily
Auto-create alerts if due or overdue
Avoid duplicates
Mark alerts “Resolved” when maintenance is done
🧱 2. Database Setup
🔹 Equipment Table
Ensure your table includes:
ALTER TABLE equipment
ADD COLUMN last_maintenance_date DATE,
ADD COLUMN next_due_date DATE,
ADD COLUMN status TEXT CHECK (status IN ('Active','Under Maintenance','Decommissioned')) DEFAULT 'Active';
🔹 Maintenance Alerts Table
If not created yet:
CREATE TABLE maintenance_alerts (
  id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  equipment_id UUID REFERENCES equipment(id),
  facility_id UUID REFERENCES facilities(id),
  location_id UUID REFERENCES locations(id),
  severity TEXT CHECK (severity IN ('Info','Warning','Critical')) DEFAULT 'Info',
  title TEXT NOT NULL,
  message TEXT NOT NULL,
  due_date DATE,
  triggered_at TIMESTAMP DEFAULT NOW(),
  status TEXT CHECK (status IN ('Open','Acknowledged','Resolved')) DEFAULT 'Open',
  resolved_at TIMESTAMP
);
🧠 3. Alert-Generation Logic
Run this as a daily cron job (midnight) or background service.
Example (Python / FastAPI + APScheduler)
from datetime import date
from database import query, execute

def generate_maintenance_alerts():
    today = date.today()
    equipment_list = query("""
        SELECT id, name, facility_id, location_id, next_due_date
        FROM equipment
        WHERE status != 'Decommissioned'
    """)

    for e in equipment_list:
        if not e["next_due_date"]:
            continue

        days_left = (e["next_due_date"] - today).days

        # Determine alert severity
        if days_left <= 0:
            severity = "Critical"
            title = f"{e['name']} maintenance overdue!"
            message = f"{e['name']} in location {e['location_id']} is {abs(days_left)} day(s) overdue."
        elif days_left <= 7:
            severity = "Warning"
            title = f"Maintenance due soon for {e['name']}"
            message = f"Maintenance is due in {days_left} day(s)."
        else:
            continue

        # Avoid duplicate active alerts
        existing = query("""
            SELECT id FROM maintenance_alerts
            WHERE equipment_id=%s AND status='Open' AND severity=%s
        """, [e["id"], severity])

        if not existing:
            execute("""
                INSERT INTO maintenance_alerts
                (equipment_id, facility_id, location_id, severity, title, message, due_date)
                VALUES (%s,%s,%s,%s,%s,%s,%s)
            """, [
                e["id"], e["facility_id"], e["location_id"],
                severity, title, message, e["next_due_date"]
            ])
✅ Runs automatically
✅ Generates Critical for overdue items and Warning for those due soon
✅ Ignores already handled alerts
🕐 4. Automation (Node.js alternative)
If you’re using Node.js and PostgreSQL:
import cron from 'node-cron';
import db from './db.js';
import dayjs from 'dayjs';

cron.schedule('0 0 * * *', async () => {
  const today = dayjs();
  const { rows: equipment } = await db.query(
    `SELECT id, name, facility_id, location_id, next_due_date
     FROM equipment WHERE status != 'Decommissioned'`
  );

  for (const e of equipment) {
    if (!e.next_due_date) continue;
    const daysLeft = dayjs(e.next_due_date).diff(today, 'day');

    let severity, title, message;
    if (daysLeft <= 0) {
      severity = 'Critical';
      title = `Overdue maintenance: ${e.name}`;
      message = `${e.name} is ${Math.abs(daysLeft)} day(s) overdue.`;
    } else if (daysLeft <= 7) {
      severity = 'Warning';
      title = `Maintenance due soon: ${e.name}`;
      message = `Maintenance due in ${daysLeft} day(s).`;
    } else continue;

    const exists = await db.query(
      'SELECT id FROM maintenance_alerts WHERE equipment_id=$1 AND status=$2 AND severity=$3',
      [e.id, 'Open', severity]
    );
    if (!exists.rowCount) {
      await db.query(
        'INSERT INTO maintenance_alerts (equipment_id, facility_id, location_id, severity, title, message, due_date) VALUES ($1,$2,$3,$4,$5,$6,$7)',
        [e.id, e.facility_id, e.location_id, severity, title, message, e.next_due_date]
      );
    }
  }
});
Runs nightly and keeps your alerts current.
🔄 5. Auto-Resolution When Maintenance Completes
Whenever a maintenance task is marked complete:
def resolve_alerts(equipment_id):
    execute("""
        UPDATE maintenance_alerts
        SET status='Resolved', resolved_at=NOW()
        WHERE equipment_id=%s AND status='Open'
    """, [equipment_id])
✅ Alerts automatically close once equipment maintenance is logged.

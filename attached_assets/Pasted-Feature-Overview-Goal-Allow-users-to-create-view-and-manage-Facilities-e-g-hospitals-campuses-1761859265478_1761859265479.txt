Feature Overview
Goal:
Allow users to create, view, and manage Facilities (e.g., hospitals, campuses) and their internal Locations (departments, rooms, labs), and assign each Equipment item to a specific location within a facility.
This creates hierarchy and improves filtering, reporting, and alert targeting.
Facility → Locations → Equipment → Maintenance Tasks
🏗️ 2. Backend Implementation
Database Tables
CREATE TABLE facilities (
  id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  name TEXT NOT NULL,
  code TEXT UNIQUE,
  address TEXT,
  contact_person TEXT,
  phone TEXT,
  created_at TIMESTAMP DEFAULT NOW()
);

CREATE TABLE locations (
  id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  facility_id UUID REFERENCES facilities(id) ON DELETE CASCADE,
  name TEXT NOT NULL,
  floor TEXT,
  room TEXT,
  department TEXT,
  notes TEXT,
  UNIQUE (facility_id, name)
);

ALTER TABLE equipment
  ADD COLUMN facility_id UUID REFERENCES facilities(id),
  ADD COLUMN location_id UUID REFERENCES locations(id);
API Endpoints
Method	Endpoint	Description
GET	/facilities	List all facilities
POST	/facilities	Create a new facility
PATCH	/facilities/:id	Update facility details
DELETE	/facilities/:id	Delete a facility
GET	/facilities/:id/locations	Get all locations under a facility
POST	/facilities/:id/locations	Add a location to a facility
PATCH	/locations/:id	Update location
DELETE	/locations/:id	Delete location
Equipment Integration
Method	Endpoint	Description
GET	/equipment?facility_id=&location_id=	Filter by facility/location
POST	/equipment	Create equipment linked to facility/location
PATCH	/equipment/:id	Update assigned facility/location

task.facility_id = equipment.facility_id

📊 5. Dashboard & Reports
Add facility-based grouping and metrics:
“Tasks per Facility”
“Overdue Maintenance by Department”
“Equipment Distribution by Location”
Example SQL:
SELECT f.name AS facility, COUNT(e.id) AS equipment_count
FROM equipment e
JOIN facilities f ON e.facility_id = f.id
GROUP BY f.name;

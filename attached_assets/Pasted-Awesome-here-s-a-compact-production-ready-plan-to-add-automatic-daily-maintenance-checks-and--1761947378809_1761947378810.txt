# 1) Minimal schema & indexes

```sql
-- Ensure equipment has a schedule
ALTER TABLE equipment
  ADD COLUMN IF NOT EXISTS maintenance_frequency_days INT,   -- e.g., 30
  ADD COLUMN IF NOT EXISTS last_maintenance_date DATE,
  ADD COLUMN IF NOT EXISTS next_due_date DATE,
  ADD COLUMN IF NOT EXISTS status TEXT
    CHECK (status IN ('Active','Under Maintenance','Decommissioned'))
    DEFAULT 'Active';

-- Alerts table
CREATE TABLE IF NOT EXISTS maintenance_alerts (
  id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  equipment_id UUID NOT NULL REFERENCES equipment(id),
  facility_id UUID REFERENCES facilities(id),
  location_id UUID REFERENCES locations(id),
  severity TEXT CHECK (severity IN ('Info','Warning','Critical')) NOT NULL,
  title TEXT NOT NULL,
  message TEXT NOT NULL,
  due_date DATE NOT NULL,
  status TEXT CHECK (status IN ('Open','Acknowledged','Resolved','Escalated')) DEFAULT 'Open',
  escalation_level INT DEFAULT 0,
  triggered_at TIMESTAMPTZ DEFAULT NOW(),
  acknowledged_at TIMESTAMPTZ,
  resolved_at TIMESTAMPTZ,
  -- Single active thread per equipment
  UNIQUE (equipment_id, status) WHERE status IN ('Open','Escalated')
);

-- Helpful indexes
CREATE INDEX IF NOT EXISTS idx_equipment_due ON equipment (next_due_date) WHERE status <> 'Decommissioned';
CREATE INDEX IF NOT EXISTS idx_alerts_status_sev ON maintenance_alerts (status, severity);
```

# 2) Daily maintenance checker (Asia/Kuwait)

**Run every day ~02:00 Asia/Kuwait** to avoid midnight boundary issues.

## Option A — Node.js (Postgres + node-cron)

```js
// jobs/dailyMaintenance.js
import cron from 'node-cron';
import dayjs from 'dayjs';
import utc from 'dayjs/plugin/utc.js';
import tz from 'dayjs/plugin/timezone.js';
import db from '../db.js'; // pg Pool
dayjs.extend(utc); dayjs.extend(tz);

const ZONE = 'Asia/Kuwait';

cron.schedule('0 2 * * *', async () => {
  const today = dayjs().tz(ZONE).startOf('day');

  // advisory lock to prevent double-execution across instances
  const { rows: lock } = await db.query('SELECT pg_try_advisory_lock($1) AS ok', [748001]);
  if (!lock[0].ok) return;

  try {
    const { rows: eqs } = await db.query(`
      SELECT id, name, facility_id, location_id, next_due_date
      FROM equipment
      WHERE status <> 'Decommissioned' AND next_due_date IS NOT NULL
    `);

    for (const e of eqs) {
      const daysLeft = dayjs(e.next_due_date).diff(today, 'day');
      let sev, title, msg;

      if (daysLeft <= 0) {
        sev = 'Critical';
        title = `Overdue maintenance: ${e.name}`;
        msg = `${e.name} is ${Math.abs(daysLeft)} day(s) overdue.`;
      } else if (daysLeft <= 7) {
        sev = 'Warning';
        title = `Maintenance due soon: ${e.name}`;
        msg = `Due in ${daysLeft} day(s).`;
      } else if (daysLeft <= 14) {
        sev = 'Info';
        title = `Upcoming maintenance: ${e.name}`;
        msg = `Planned in ${daysLeft} day(s).`;
      } else continue;

      // Upsert a single active alert per equipment
      await db.query(`
        INSERT INTO maintenance_alerts (equipment_id, facility_id, location_id, severity, title, message, due_date)
        VALUES ($1,$2,$3,$4,$5,$6,$7)
        ON CONFLICT (equipment_id, status) WHERE status IN ('Open','Escalated')
        DO UPDATE SET severity=EXCLUDED.severity, title=EXCLUDED.title,
                      message=EXCLUDED.message, due_date=EXCLUDED.due_date
      `, [e.id, e.facility_id, e.location_id, sev, title, msg, e.next_due_date]);
    }
  } finally {
    await db.query('SELECT pg_advisory_unlock($1)', [748001]);
  }
}, { timezone: ZONE });
```

## Option B — Python (APScheduler + psycopg)

```python
# jobs/daily_maintenance.py
from datetime import date
import psycopg

def run_daily_checks():
    today = date.today()
    with psycopg.connect() as conn, conn.cursor() as cur:
        cur.execute("SELECT pg_try_advisory_lock(%s)", (748001,))
        if not cur.fetchone()[0]:
            return

        try:
            cur.execute("""
              SELECT id, name, facility_id, location_id, next_due_date
              FROM equipment
              WHERE status <> 'Decommissioned' AND next_due_date IS NOT NULL
            """)
            for eid, name, fid, lid, due in cur.fetchall():
                days_left = (due - today).days
                if   days_left <= 0: sev,title,msg = 'Critical', f'Overdue maintenance: {name}', f'{name} is {abs(days_left)} day(s) overdue.'
                elif days_left <= 7: sev,title,msg = 'Warning', f'Maintenance due soon: {name}', f'Due in {days_left} day(s).'
                elif days_left <= 14: sev,title,msg = 'Info', f'Upcoming maintenance: {name}', f'Planned in {days_left} day(s).'
                else: continue

                cur.execute("""
                  INSERT INTO maintenance_alerts (equipment_id, facility_id, location_id, severity, title, message, due_date)
                  VALUES (%s,%s,%s,%s,%s,%s,%s)
                  ON CONFLICT (equipment_id, status) WHERE status IN ('Open','Escalated')
                  DO UPDATE SET severity=EXCLUDED.severity, title=EXCLUDED.title,
                                message=EXCLUDED.message, due_date=EXCLUDED.due_date
                """, (eid, fid, lid, sev, title, msg, due))
            conn.commit()
        finally:
            cur.execute("SELECT pg_advisory_unlock(%s)", (748001,))
```

# 3) Auto-resolution hook (when a task completes)

Call this in your “complete maintenance task” service:

```sql
-- Close any active alert threads for this equipment
UPDATE maintenance_alerts
SET status='Resolved', resolved_at=NOW()
WHERE equipment_id = $1 AND status IN ('Open','Escalated','Acknowledged');

-- Roll schedule (example using frequency days)
UPDATE equipment
SET last_maintenance_date = CURRENT_DATE,
    next_due_date = CURRENT_DATE + (maintenance_frequency_days || ' days')::interval
WHERE id = $1;
```

# 4) Optional hourly escalation

```sql
-- Escalate open critical alerts lingering ≥ 3 days past due
UPDATE maintenance_alerts a
SET status='Escalated', escalation_level = escalation_level + 1
WHERE a.status='Open'
  AND a.severity='Critical'
  AND CURRENT_DATE - a.due_date >= 3;
```

# 7) Reliability notes

* Use **pg advisory lock** (or Redis lock) to keep only one runner active.
* Compare **date to date** (`CURRENT_DATE` vs `next_due_date`) to avoid off-by-one.
* Schedule at **02:00 Asia/Kuwait**.
* Make the job **idempotent** (UPSERT as shown).

---

# 8) Tests (must pass)

**Unit**

* `next_due_date = today+10` → no alert.
* `+14..+8` → Info; `+7..+1` → Warning; `<=0` → Critical.
* Re-run job: no duplicates; record updates in place.

**Integration**

1. Seed equipment: `next_due_date=today+3`.
2. Run job → **Warning/Open** alert.
3. Move clock to due+2, run job → alert becomes **Critical** (same row updated).
4. Complete maintenance → alert **Resolved**, equipment `next_due_date` rolled.

**API**

* `PATCH /api/alerts/:id/ack` sets status & timestamp.
* Filters by facility/severity/status work.

What happened

🧠 2. Backend Improvements (Handling Logic)
A. Add Rich Alert Schema
Enhance your alerts table to store all contextual data:
CREATE TABLE alerts (
  id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  entity_type TEXT CHECK (entity_type IN ('Equipment', 'MaintenanceTask', 'Contract')),
  entity_id UUID NOT NULL,
  severity TEXT CHECK (severity IN ('Info', 'Warning', 'Critical')) DEFAULT 'Info',
  title TEXT NOT NULL,
  message TEXT NOT NULL,
  facility_id UUID REFERENCES facilities(id),
  location_id UUID REFERENCES locations(id),
  status TEXT CHECK (status IN ('Open', 'Acknowledged', 'Resolved', 'Escalated')) DEFAULT 'Open',
  triggered_at TIMESTAMP DEFAULT NOW(),
  acknowledged_by UUID REFERENCES users(id),
  acknowledged_at TIMESTAMP,
  resolved_at TIMESTAMP,
  escalation_level INT DEFAULT 0
);
B. Intelligent Alert Generation
Generate alerts based on thresholds:
Equipment due for maintenance in 7 days → Warning
Overdue > 3 days → Critical
Contract expired → Info
Merge duplicate alerts (prevent spam):
key = f"{entity_type}:{entity_id}:{severity}"
if not redis.exists(key):
    create_alert(...)
    redis.setex(key, timedelta(hours=6), "sent")
Auto-escalate critical alerts to supervisors:
if alert.severity == 'Critical' and days_overdue > 7:
    alert.status = 'Escalated'
    notify_supervisor(alert)
C. Alert Lifecycle States
State	Description	Trigger
Open	New alert created	Maintenance overdue, task missed
Acknowledged	User viewed it	User clicks “Acknowledge”
Resolved	Issue fixed	Related task marked completed
Escalated	Ignored critical alert	System auto-escalates

🧾 C. AlertCard Component (Detailed View)
export function AlertCard({ alert, onAcknowledge }) {
  const color =
    alert.severity === 'Critical' ? 'border-red-500 bg-red-50' :
    alert.severity === 'Warning' ? 'border-yellow-500 bg-yellow-50' :
    'border-blue-500 bg-blue-50';

  return (
    <div className={`p-4 mb-3 rounded-xl border-l-4 ${color}`}>
      <div className="flex justify-between items-center">
        <h4 className="font-semibold">{alert.title}</h4>
        <StatusBadge status={alert.severity} />
      </div>
      <p className="text-sm text-gray-600 mt-1">{alert.message}</p>
      <p className="text-xs text-gray-500 mt-1">
        {alert.facility_name} • {alert.location_name} • {formatDate(alert.triggered_at)}
      </p>

      <div className="flex gap-2 mt-2">
        <button
          className="text-blue-600 text-xs"
          onClick={() => openRelated(alert.entity_type, alert.entity_id)}>
          View Details
        </button>
        {alert.status === 'Open' && (
          <button
            className="text-green-600 text-xs"
            onClick={() => onAcknowledge(alert.id)}>
            Acknowledge
          </button>
        )}
      </div>
    </div>
  );
}

🔄 5. Automatic Resolution Sync
When a related maintenance task is marked Completed, automatically resolve associated alerts:
UPDATE alerts
SET status='Resolved', resolved_at=NOW()
WHERE entity_type='MaintenanceTask' AND entity_id=:task_id AND status!='Resolved';
